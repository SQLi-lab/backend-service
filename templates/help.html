Руководство пользователя

Цель лабораторной работы: проэксплуатировать уязвимости SQL-Injection на сайте и найти 3 флага.

  • Студент может создать уникальную лабораторную работу, ей присваиваются имя и uuid.
  • После завершения создания будет доступна ссылка на уязвимый сайт.
  • На выполнение работы дается 3 часа, после чего лабораторная удаляется.
  • Каждый раз генерируется случайный вариант из разных предметных областей (типов сайтов).
  • Каждому студенту присваиваются свои уникальные флаги, которые необходимо найти на сайте: ibks1_{...}, ibks2_{...}, ibks3_{...}.
  • Ответ на лабораторную работу имеет вид: secret_"содержимое_флагов_по_очереди".
Пример:
  • Флаги: ibks1_{4564124c79b0b}, ibks2_{64e1cdb406a36}, ibks3_{fa435f3ae179e}.
  • Ответ: secret_4564124c79b0b64e1cdb406a36fa435f3ae179e.
После ввода правильного ответа лабораторная работа считается выполненной.
Если студент выполнил хотя бы одну работу, то при дальнейшем создании работ ему будет выделено только 30 минут на выполнение.
Для выполнения работы рекомендуется использовать не только браузер, но и инструементы Postman, BurpSuit, ZAP и т.п.

Система разработана с использованием языков программирования Python 3.12, Go 1.23 и фреймворка React 18.3.1.
Для развертывания и управления используются Docker и Ansible.
В качестве поддерживающих компонентов инфраструктуры применяются Postgres, Redis и Traefik.
